Ruminations about programming languages

April 12, 2019

Today I’ve come across a post in Quora about the “end of Java”, kind of funny these doomsday predictions regarding programming languages.

I’ve starting to learn programming somewhere around the late 80’s when I was a kid, at the time I’ve learn basic … it was the most common language for kids who owned “PC’s” with 48k of memory. Basic programs tended to be quite difficult to read and one thing that you learn from experience ( sometimes the hard way ) is that it is generally much difficult to maintain and evolve computer software than to initially write it, thus putting effort in to make it clean and adaptable pays off.

Eventually I’ve left Basic for more structured languages like C/C++ and Java, I’ve been using Java as my “main” programming language for many years now. I’m I worried that it is dying ? well, a short answer is no, let’s think about it.

Definitely Java is getting mature now, although the latest features are interesting the level of innovation in recent years is not what it used to be. But that doesn’t mean it is going anywhere, Basic was born in 1964 an is still used, many people still make a very decent living programming COBOL that is even older ( from 1959 ), by comparison Java is a “millennial” language born in 1995.

The amount of code written in Java is huge, much of this code is not to be replaced anytime soon … write software is a costly and difficult craft. Today is still the 2n most active language on [github] (http://githut.info). Funny some of the more “trendy” languages like Python or GO still lag behind.

Other important factor to consider is that companies tend to stick with their language choices, changing development languages is a costly issue thus “Java shops” tend to continue with Java … there are many “Java shops” around and these are not going anywhere.

Although with modern paradigms like containers & microservices it is less of a problem to integrate several programming languages in a solution this is not always a good idea. One obvious issue is the maintenance problem, generally you have a large team to develop a new solution but only a few people are “left behind” when it comes to maintenance phase, how expensive it is to maintain if it is written in an mixture of Java, Java Script C, C++, Python, Shell, Go, Python and whatever the developers found fancy at the time ?

That said it is not strange for large organizations to use a few different “related” programming languages for different purposes – for example usage of Java in the backend and JavaScript in the frontend is typical these days. The reality is that all the languages have their strong & weak points – but if you are developing really big solutions with complex requirements like high availability, high security, disaster recovery, etc… you better stick to wide used mature languages as the newer “trendy” languages not always solve these problems in the first releases.

In the end choosing programming languages is not that much different from choosing “normal” languages – put a bunch of Europeans techies from different countries in a room and they will switch to English is no time, not because it is better or has more “features” but because is the common language everyone in the room knows. Brexit or No-Brexit that’s not about to change…I guess the end of Java is not near as well.
